---

# 1. Objective

The purpose of this test plan is to verify that the core functionalities of the OrangeHRM Demo application work correctly and meet the expected business requirements before release.

---

# 2. Scope

## In Scope

- User Login
- User Logout
- Dashboard
- Employee Management (PIM)

## Out of Scope

- Recruitment Module
- Payroll Module
- Mobile Application

---

# 3. Test Strategy

The testing activities will include:

- Functional Testing
- Smoke Testing
- Positive Testing
- Negative Testing

Testing will be performed manually using a web browser.

---

# 4. Entry Criteria

Testing can begin when:

- Application is deployed successfully.
- Test environment is available.
- Test account is ready.
- Business requirements are available.

---

# 5. Exit Criteria

Testing will be completed when:

- All planned test cases have been executed.
- No Critical or High severity defects remain open.
- Test Summary Report has been completed.

---

# 6. Test Environment

| Item | Value |
|------|-------|

---

# 7. Risks

Potential risks during testing:

- Test environment unavailable
- Requirement changes
- Unstable internet connection

# 8. Deliverables

This testing project will produce:

- Test Plan

- Test Case
